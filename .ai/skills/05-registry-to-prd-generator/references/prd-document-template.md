# 页面级 PRD 文档模板

生成页面级 Markdown PRD 时，使用下面结构。

## 模板

```md

- 页面名称：
- 所属模块：
- 页面路由/组件：
- 相关代码文件：

### 1.2 上级页面/上游入口
- 上级页面：
- 入口位置：
- 进入方式：
- 传入数据：
- 依赖状态：

### 1.3 下级页面/下游流程
- 下级页面/下游流程：
- 触发方式：
- 传出数据：
- 后续状态：
- 返回逻辑：

### 1.4 范围边界摘要
- 本文档覆盖：
- 本文档不覆盖：
```

## 上游/下游填写规则

### 当前页面

从注册表读取：

- `pageName`
- `module`
- `route`
- `relatedFiles`

### 上级页面/上游入口

优先从代码调用关系判断：

- 当前组件在哪些页面被 import 和渲染。
- 父页面通过什么状态打开当前组件。
- 父页面给当前组件传了什么 props。
- 当前组件是否依赖父页面回调。

如果是页面路由，说明它从哪个入口进入。

### 下级页面/下游流程

优先从操作说明和代码事件判断：

- 点击哪个按钮触发下游。
- 是否调用父组件回调。
- 是否打开弹窗。
- 是否跳转路由。
- 是否写入缓存或传出数据。

如果无明确下游，写“未在当前代码或注册表中确认明确下游流程”。

## 过滤规则

默认过滤：

- 无额外权限限制
- 无额外数据流转
- 无异常场景
- 本对象无操作入口
- 本对象仅展示
- 空字符串
- 空数组

## sourceType 中文映射

- `code`：代码事实
- `decision`：确认决策
- `code+decision`：代码事实 + 确认决策
