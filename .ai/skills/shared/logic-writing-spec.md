# 业务逻辑写作规范

本规范约束需求注册表、右侧 PRD 阅读面板、悬浮业务逻辑面板、Markdown PRD 中“显示说明”和“操作说明”的写法。

## 总原则

- 显示说明只写“用户在页面上看见什么”。
- 操作说明只写“用户怎么操作，系统怎么处理”。
- 两者不能重复。
- 注册表可以保留结构化兜底说明，方便机器判断信息完整性。
- 面向用户阅读的 PRD 面板和 Markdown PRD 不展示空兜底信息，避免增加阅读成本。

## 显示说明检查框架

显示说明只回答：页面上让用户看见了什么。

### 1. 对象识别层

说明这个需求对应哪个对象：

- 字段
- 文案
- 按钮
- 区域
- 弹窗
- Tab
- 步骤
- 列表/卡片
- 状态提示

### 2. 默认展示层

说明默认情况下用户看到什么：

- 默认文案
- 默认值
- 默认选中项
- 默认按钮状态
- 默认是否展开
- 默认是否显示

### 3. 状态展示层

说明不同状态下如何展示：

- 正常态
- 禁用态
- 加载态
- 空态
- 成功态
- 失败态
- 已选中/未选中
- 已展开/已收起
- 已完成/未完成
- 已读/未读
- 进行中/已结束

### 4. 条件展示层

说明在什么条件下显示、隐藏、切换：

- 角色条件
- 数据条件
- 流程步骤条件
- Tab 条件
- 弹窗开关条件
- 是否已有数据
- 是否满足校验
- 是否在本次迭代范围

### 5. 内容结构层

说明展示内容由哪些部分组成：

- 标题
- 主文案
- 副文案
- 图标
- 标签
- 数值
- 列表项
- 操作按钮
- 提示信息

### 6. 关系位置层

说明它和其他对象的展示关系：

- 属于哪个区域
- 是否和其他组件互斥
- 是否跟随某个组件出现
- 是否覆盖页面
- 是否挤压页面布局
- 是否在弹窗/悬浮层/侧边栏里

### 7. 可读性与反馈层

说明用户是否能理解当前状态：

- 是否有清晰提示
- 禁用原因是否可见
- 失败原因是否可见
- 空状态是否说明下一步
- 当前选中项是否明显
- 当前定位/高亮是否明显

## 操作说明检查框架

操作说明只回答：用户做了什么，系统怎么处理。

### 1. 操作入口层

说明用户能通过什么入口操作：

- 点击
- 输入
- 选择
- 拖拽
- 上传
- 关闭
- 切换
- 滚动
- hover
- 快捷操作

### 2. 操作前置条件层

说明什么时候可以操作：

- 角色权限
- 数据是否存在
- 流程是否到达某步骤
- 是否已选择必填项
- 是否处于加载中
- 是否处于禁用态
- 是否在本次迭代范围

### 3. 操作触发层

说明操作发生后立即触发什么：

- 更新本地 state
- 调用父组件回调
- 打开弹窗
- 关闭面板
- 切换 Tab
- 跳转页面
- 追加消息
- 发起接口
- 写入缓存

### 4. 数据流转层

说明数据从哪里来、到哪里去：

- props 来源
- 用户输入
- 本地 state
- sessionStorage/localStorage
- API 请求/响应
- 父子组件传参
- 文件对象
- mock 数据
- 最终写入位置

### 5. 权限与边界层

说明谁能操作、谁不能操作：

- 教师/学生/管理员
- 数据所有者
- 已上线能力
- 本期不做能力
- 当前页面不负责的能力
- 只读场景
- 禁用场景

### 6. 异常与兜底层

说明异常时怎么处理：

- 输入为空
- 文件不支持
- 接口失败
- 超时
- 重复点击
- 数据缺失
- 状态不同步
- 关闭中断
- 刷新丢失
- 回调不存在
- 没有权限
- 结果为空

### 7. 操作后状态层

说明操作完成后页面变成什么：

- 按钮隐藏/显示
- 面板打开/关闭
- 消息追加
- 状态变为成功/失败
- 列表刷新
- 高亮变化
- 跳转后恢复
- 缓存被清理
- 用户能否继续下一步

### 8. 验收判断层

说明如何判断这个操作是对的：

- 用户能看到什么结果
- 系统状态是否变化
- 是否调用正确回调
- 是否阻止非法操作
- 是否保留/清理正确数据
- 异常提示是否出现

## 结构化兜底说明

生成注册表时，如果某个层面没有额外逻辑，可以在结构化字段里写清楚，避免误判为遗漏，例如：

- 无额外权限限制
- 无额外数据流转
- 无异常场景
- 本对象无操作入口
- 本对象仅展示

## 面向用户渲染规则

右侧 PRD 阅读面板、悬浮业务逻辑面板、Markdown PRD 默认不展示上面的空兜底说明。

渲染时应过滤掉下列无信息量内容：

- “无额外权限限制”
- “无额外数据流转”
- “无异常场景”
- “本对象无操作入口”
- “本对象仅展示”
- 空字符串
- 空数组

只有当用户明确要求“展示完整字段”或“展示机器原始注册表”时，才显示这些兜底内容。

## 质量检查

- 如果显示说明里出现“点击后、调用、跳转、写入、权限、异常、失败”等操作词，要检查是否混入操作说明。
- 如果操作说明里大段重复“页面显示、展示文案、按钮显示为”等内容，要检查是否重复显示说明。
- 如果权限、数据流转、异常情况确实没有额外内容，可以在注册表保留兜底说明，但用户阅读面板不要展示。
