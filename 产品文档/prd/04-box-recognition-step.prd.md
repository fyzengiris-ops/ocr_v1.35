# 框选识别步骤 PRD

## 1. 页面范围

### 1.1 当前页面
- 页面名称：框选识别步骤
- 所属模块：识别作业资料
- 页面路由/组件：/homework → 识别作业资料 → 框选&识别步骤
- 相关代码文件：
- `src/components/UploadQuestionDialog.tsx`
- `src/app/homework/page.tsx`

### 1.2 上级页面/上游入口
- 上级页面：资料场景方式选择步骤。
- 入口位置：用户完成识别模式选择，必要时完成文件用途分配。
- 进入方式：上传录题弹窗切换到「框选&识别」步骤。
- 传入数据：接收资料图片、识别模式、文件用途、学段学科和页码范围。
- 依赖状态：需要已有资料页面；自动切题中部分按钮不可操作。

### 1.3 下级页面/下游流程
- 下级页面/下游流程：核查题目信息步骤。
- 触发方式：用户选中题目框后点击「开始识别」。
- 传出数据：传出框选区域、识别模式、学段学科、识别后的题目、答案和解析数据。
- 后续状态：识别完成后右侧进入题目核查；识别过程中可暂停、继续或取消。
- 返回逻辑：用户可重新上传回到资料入口，或通过步骤条确认返回模式选择。

### 1.4 范围边界摘要
- 本文档覆盖：记录框选识别步骤页面的完整页面逻辑，覆盖统计信息栏、资料选框呈现、操作按钮、右侧空状态、文件用途弹窗、自动切题、继续/重新上传等交互边界。
- 本文档不覆盖：无额外排除项。

## 2. 功能概述

记录框选识别步骤页面的完整页面逻辑，覆盖统计信息栏、资料选框呈现、操作按钮、右侧空状态、文件用途弹窗、自动切题、继续/重新上传等交互边界。本文档按单页面/单流程沉淀，业务逻辑、验收标准和来源追溯均来自需求注册表。

## 3. 业务逻辑说明

### BOX_STEP-001 统计信息栏：已框选/已选中/一键清空/全选

来源：代码事实 + 确认决策

#### 显示说明

页面左侧资料区上方显示统计信息栏。左侧为操作提示文案「若系统框选有误或漏框选，可在资料上，以按住鼠标并拖动的方式框选区域」。右侧依次显示「已框选」数字卡片（绿色大号数字）、「已选中」数字卡片（蓝色大号数字）、「一键清空」按钮（有框时可用，hover变红）、「全选」复选框。自动切题中新文件时，数字和按钮变为灰色不可操作；切题完成后恢复。

#### 操作说明

1、首次自动切题完成后，所有框默认选中。
2、继续上传追加的新文件切题完成后，只默认选中新生成的框，保留原有框选状态。
3、无框时「一键清空」和「全选」均置灰不可操作。
4、点击「一键清空」弹出确认弹窗：「确认清空所有切题框吗？」

- 后续流程：框选数据来源于腾讯云自动切题服务或用户手动画框，选中状态影响「开始识别」按钮是否可用

- 异常边界：自动切题失败时，原左侧显示的文案「若系统框选有误或漏框选，可在资料上，以按住鼠标并拖动的方式框选区域」，替换为「未检测到内容区域，可手动画框」

#### 验收标准

1. 已框选数字正确显示
2. 已选中数字正确显示
3. 一键清空弹窗确认后可清空
4. 全选可一键选中/取消所有框
5. 切题中按钮正确置灰

### BOX_STEP-002 开始识别按钮

来源：代码事实 + 确认决策

#### 显示说明

按钮位于工具栏最右侧，当有选框选中，且无其他识别服务在进行中时，按钮高亮。按钮左侧显示当前识别方式标签（仅识别题目/识别题目和答案）。按钮文案为「开始识别」，有选中框时显示数量如「开始识别(3)」。

#### 操作说明

1、点击「开始识别」后，系统根据已选中的框裁剪对应图片区域，调用 AI 进行题目内容识别。
2、仅识别题目模式下，所有框按题目框处理，右边只展示题干。
3、识别题目和答案模式下，题目框走完整识别流程，答案框走答案提取流程，右边展示题干+答案/解析。
4、自动切题完成且有框时默认全选，按钮自动可用；无框时按钮置灰。

- 后续流程：将已选中的框、裁剪后的图片、识别模式和学科信息发送给 AI 识别服务

- 异常边界：识别过程中显示「暂停识别」和「取消识别」按钮。暂停后可继续，取消后保留框选状态不变仅终止识别。识别失败时保留已有框选，右侧显示错误信息。

#### 验收标准

1. 有选中框时按钮可用
2. 无选中框时按钮置灰
3. 切题中按钮置灰
4. 识别方式标签正确显示
5. 点击后进入识别流程

### BOX_STEP-003 右侧空状态引导

来源：代码事实 + 确认决策

#### 显示说明

点击开始识别前，右侧面板显示空状态引导。仅识别题目模式文案：「框选题目并选中后，点击「开始识别」，小乐会自动为您识别题目，完成识别后，您可在右侧以图片或文字的形式分模块查看每道题目」。识别题目和答案模式文案：「框选题目并选中后，点击「开始识别」，小乐会为您自动识别题目，并进行题目和答案的自动匹配。完成识别后，您可在右侧以图片或文字的形式分模块查看每道题的题干与答案/解析」。有选中框时下方显示「已选中 X 个识别框」。

#### 操作说明

引导文案和已选中数量为纯展示，不可交互。仅用于帮助用户理解下一步操作。

#### 验收标准

1. 两种模式文案不同
2. 已选中数量正确
3. 与操作按钮状态一致

### BOX_STEP-004 文件用途分配弹窗（识别题目和答案模式）

来源：代码事实 + 确认决策

#### 显示说明

当用户在步骤1选择的是识别题目和答案模式，进入到步骤2之后，用户又点击继续上传，完成文件上传后，弹出「指定文件用途」弹窗。每份文件显示名称和「题目文件」「答案文件」两个按钮。底部有「取消」和「确认，开始处理」按钮。当用户在步骤1只上传了一份文件，且选择的是识别题目和答案模式，进入到步骤2后，用户又点击继续上传，完成文件上传后，弹出的「指定文件用途」弹窗，显示的提示文案为「检测到你上传了X个文件，请指定文件用途」；当用户在步骤1只上传了多份文件，且选择的是识别题目和答案模式，进入到步骤2后，用户又点击继续上传，完成文件上传后，弹出的「指定文件用途」弹窗，显示的提示文案为「检测到你新上传了X个文件，请指定文件用途」。

#### 操作说明

1、每份文件独立选择文件用途，互不影响。
2、点击已选中的文件用途按钮可取消选择，恢复为未分配状态。
3、所有文件都分配文件用途后，「确认，开始处理」按钮高亮可点击。
4、点击确认后进入框选识别页面。
5、取消或✕关闭弹窗，同时取消已选的识别模式。

- 后续流程：文件用途分配结果传递给后续流程，决定文件标签切换和显示顺序

- 异常边界：关闭弹窗时如有未分配文件，直接关闭不提示

#### 验收标准

1. 文件独立选择文件用途
2. 点击已选可取消
3. 继续上传后仅显示新文件
4. 文案按场景正确切换

### BOX_STEP-005 继续上传按钮

来源：代码事实 + 确认决策

#### 显示说明

按钮位于工具栏，绿色文字样式。悬停提示「继续上传资料」。

#### 操作说明

1、点击继续上传后，打开文件选择弹窗，允许追加新文件。
2、仅识别题目模式：追加后直接开始切题，不弹任何弹窗。
3、识别题目和答案模式：追加后如有新文件形成多文件场景，弹出文件用途分配弹窗。
4、追加后保留已有框选和选中状态，仅对新页面自动切题。
5、自动切题中不允许继续上传，点击时显示提示文案。

- 后续流程：新文件追加到现有文件列表末尾

- 异常边界：正在处理文件时点击，临时显示「正在处理文件，请稍后再继续上传」3秒后消失

#### 验收标准

1. single模式不弹窗
2. stepwise模式弹文件用途弹窗
3. 切题中置灰不可点击
4. 已有框选保留

### BOX_STEP-006 重新上传按钮与确认弹窗

来源：代码事实 + 确认决策

#### 显示说明

按钮位于继续上传按钮左侧，灰色文字样式。点击弹出确认弹窗：「重新上传将清空当前资料、框选和识别方式，是否继续？」。弹窗有「取消」和「确认」按钮。

#### 操作说明

1、点击重新上传弹出确认弹窗。
2、确认后清空所有数据（框选、识别模式、文件用途、识别结果），关闭当前弹窗回到主页上传文件的弹窗页面（AI小乐面板为弹窗背景的页面）。
3、用户可直接进行上传文件的操作，无需再点击AI小乐面板里的「帮我识别作业资料」的快捷按钮。

- 后续流程：确认后清空所有流程数据，关闭弹窗

- 异常边界：确认弹窗可取消，取消后保持当前状态不变

#### 验收标准

1. 点击弹出确认弹窗
2. 确认后回到主页
3. 取消保持当前状态
4. review步骤不显示

### BOX_STEP-007 暂停/取消识别按钮

来源：代码事实 + 确认决策

#### 显示说明

点击开始识别后，右侧面板显示进度提示和两个控制按钮。「暂停识别」（橙色）点击后变为「继续识别」（绿色），spinner停止旋转。「取消识别」（灰色）始终可见。

#### 操作说明

1、点击「暂停识别」后，中止当前识别请求，暂停右侧处理状态，按钮变为「继续识别」。
2、点击「继续识别」后，重新提交识别请求，从暂停处继续。
3、点击「取消识别」后，中止识别请求，保留左侧框选和选中状态不变，仅终止本次识别操作。

- 后续流程：暂停时保留已识别的部分结果，继续时重新提交

- 异常边界：暂停后不创建模拟数据，不清除已有结果

#### 验收标准

1. 暂停按钮切换为继续
2. 暂停时spinner停止
3. 取消保留框选状态

### BOX_STEP-008 继续上传后自动切题提示

来源：代码事实 + 确认决策

#### 显示说明

继续上传追加新文件后，左侧资料预览区顶部显示蓝色提示条「正在处理新文件...」，带有旋转加载动画。提示条仅在追加后的自动切题期间显示（已有框 + 正在切题中），首次进入页面切题时不显示。切题完成后自动消失。

#### 操作说明

提示条为纯展示，仅用于告知用户系统正在处理新追加的文件。

#### 验收标准

1. 追加后切题时显示
2. 首次切题不显示
3. 切题完成后消失

### BOX_STEP-010 资料预览区选框层：呈现、操作与回退状态

来源：代码事实

#### 显示说明

步骤2「选择识别内容」中，左侧资料预览区按页展示上传资料。每页右上角显示「本页 X 框 / 共 Y 框」。自动切题或手动画出的识别框覆盖在资料图片上：选中框为绿色边框和浅绿色底色，左上角显示绿色圆形勾选按钮；未选中框为灰色边框和浅色底色；未识别框右上角显示黄色「待识别」标签和红色删除按钮。每个框四角显示绿色调整手柄；跨页框显示「↓ 跨页」或「跨页续 ↑」提示；用户正在拖拽画框时，页面上显示绿色虚线临时框。

#### 操作说明

1、用户在资料空白处按住鼠标并拖动，可手动画出识别框，松开后新框默认选中。
2、用户可点击左上角圆形勾选按钮切换单个框是否参与识别，也可用统计栏「全选」一次性切换所有框。
3、用户可拖动框内部移动位置，拖动四角手柄调整大小；跨页资料支持从当前页拖到下一页形成跨页框。
4、用户可点击红色删除按钮删除单个框，也可用统计栏「一键清空」删除全部框。
5、点击「开始识别」时，只有当前选中的框进入识别；已识别框如果被重新选中并调整，会按新范围重新识别。

- 后续流程：识别框来自自动切题结果或用户手动画框。框的位置、大小、所在页、选中状态和识别状态会持续留在本次识别流程中；开始识别时，系统只读取已选中的框并裁剪对应资料区域，识别完成后把题目数据展示到右侧核对区，同时取消已识别框的选中状态。用户从步骤3返回步骤2时，框选区域和已识别题目继续保留，只清空选中状态，便于重新选择或调整后再次识别。

- 异常边界：自动切题失败或未切出内容时，页面保留手动画框能力；用户取消识别时，已框选区域和选中状态不被清空；用户删除单个框或一键清空后，对应框不再参与后续识别。

#### 验收标准

1. 资料页上能看到自动切题或手动画出的识别框
2. 选中框和未选中框样式有明确区分
3. 单框勾选、删除、移动、缩放可用
4. 跨页框显示跨页提示
5. 首次进入步骤2自动框默认选中
6. 继续上传后仅新增框默认选中，原有框状态保留
7. 从步骤3返回步骤2后框保留但全部不选中
