# 核查题目信息 PRD

## 1. 页面范围

### 1.1 当前页面
- 页面名称：核查题目信息
- 所属模块：识别作业资料
- 页面路由/组件：/homework → 识别作业资料 → 核查题目信息步骤
- 相关代码文件：
- `src/components/UploadQuestionDialog.tsx`
- `src/app/homework/page.tsx`
- `src/app/api/recognize-questions/route.ts`
- `src/lib/ai-recognizer.ts`
- `src/app/paper-edit/page.tsx`

### 1.2 上级页面/上游入口
- 上级页面：框选识别步骤。
- 入口位置：开始识别完成后进入核查题目信息。
- 进入方式：系统根据识别结果切换到 review 阶段。
- 传入数据：接收已识别题目、答案、解析、子题结构、裁剪图片和匹配状态。
- 依赖状态：需要至少完成一次题目识别；识别失败时停留在框选识别阶段。

### 1.3 下级页面/下游流程
- 下级页面/下游流程：`/paper-edit` 试卷编辑页。
- 触发方式：用户点击「加入试卷」。
- 传出数据：传出父题、子题、答案、解析、填空空位答案、题型、选项数、题目图片和学段学科。
- 后续状态：通过校验或二次确认后进入试卷编辑页；弹窗状态会暂存用于返回录题。
- 返回逻辑：试卷编辑页点击「返回录题」回到 `/homework`，并可恢复上传录题弹窗。

### 1.4 范围边界摘要
- 本文档覆盖：记录核查题目信息步骤的完整页面逻辑，覆盖匹配状态提示、题目卡片展示、子题结构与编辑、答案/解析核对与关联、图片模式与编辑模式切换、加入试卷等交互边界。
- 本文档不覆盖：解答题答案/解析自动分发规则中的不可拆分情况

## 2. 功能概述

记录核查题目信息步骤的完整页面逻辑，覆盖匹配状态提示、题目卡片展示、子题结构与编辑、答案/解析核对与关联、图片模式与编辑模式切换、加入试卷等交互边界。本文档按单页面/单流程沉淀，业务逻辑、验收标准和来源追溯均来自需求注册表。

### REVIEW_STEP-001 匹配状态提示（分步模式）

来源：代码事实 + 确认决策

识别题目和答案模式下，右侧面板顶部显示匹配状态提示条：「请检查题目与答案匹配情况：共 X 道题目，已匹配答案 Y 题，待匹配答案 N 题（待匹配答案的题目，可在左侧框选答案/解析手动关联题目）」。提示条始终完整展示，不可收起或展开。

#### 操作说明

1、提示条为纯展示，帮助用户了解当前答案匹配进度。

- 后续流程：匹配统计数据来源于已识别的题目和答案数据

2. 待匹配数量正确

### REVIEW_STEP-002 题目卡片展示（仅识别题目模式）

来源：代码事实 + 确认决策

#### 显示说明

仅识别题目模式下，右侧按题号展示每道题目。每张卡片包含题号、题型、题目内容，以及图片预览区域。不单独展示答案/解析区域。如果用户框选的区域里包含答案或解析内容，由用户通过裁剪功能手动处理。

#### 操作说明

1、用户可切换图片模式查看原始截图，或编辑模式查看AI识别的文字。
2、在图片模式或编辑模式下，用户都可为每道题修改题型；均可通过点击题号快速定位到右侧对应的题目。
3、在编辑模式下，可更改AI识别的题目内容。
4、用户可使用裁剪功能截取题目图片中需要的部分。
5、用户可为复合题添加子题结构。

- 后续流程：题目数据来源于AI识别结果，用户编辑后更新本地状态，加入试卷时传递给下游

- 异常边界：识别失败时停留在框选识别阶段，不进入本页面

#### 验收标准

1. 题目卡片正确展示
2. 图片/编辑模式可切换
3. 题型和内容可编辑
4. 裁剪功能可用

#### 操作说明

4、用户可使用裁剪功能截取题目图片中需要的部分。
5、用户可为复合题添加子题结构。

- 后续流程：题目数据来源于AI识别结果和答案匹配结果，用户编辑后更新本地状态，加入试卷时传递给下游

- 异常边界：识别失败时停留在框选识别阶段

4. 解答题显示子题结构

### REVIEW_STEP-004 子题结构与编辑

来源：代码事实 + 确认决策

#### 显示说明

解答题、完形填空、阅读理解等复合题型支持子题结构。在题目卡片中显示子题列表，每个子题包含序号（①②③...）、题型选择器、答案和解析区域。当用户在第二步骤框选题目后，点击开始识别，系统在进行识别的时候，如果存在有子题的大题，需要系统把对应大题的子题结构自动添加，并且子题题型根据每个子题的题干要求自动匹配。父题显示子题匹配进度，例如「2/3 子题已匹配」。用户可点击「+子题」新增空子题。每个子题可独立编辑题型、内容、答案和解析。删除已有内容的子题时弹出确认：「该子题已有答案/解析，删除后内容将一并移除，是否继续？」

#### 操作说明

1、题干中的子题标记（如①、②）数量 ≥2 时，自动生成子题结构。
2、子题题型根据子题题干要求自动匹配，不固定继承父题题型。
3、答案/解析按子题标记自动拆分到对应子题；无法拆分时保留在父题区并显示「答案解析待人工拆分」。
4、点击「+子题」新增空子题。
5、删除子题时如有已有内容，弹窗确认后一并删除。

- 后续流程：子题数据作为题目数据的一部分，加入试卷时保留完整子题结构

- 异常边界：题干少于2个子题标记时不自动生成子题结构

#### 验收标准

1. 子题自动拆分正确
2. 子题题型自动匹配
3. 新增子题为空
4. 删除有内容的子题弹窗确认
5. 加入试卷保留子题结构

### REVIEW_STEP-005 答案关联弹窗

来源：代码事实 + 确认决策

#### 显示说明

用户在左侧框选答案区域后，弹出「请关联题号」弹窗。弹窗标题「请关联题号」，说明文案「为当前框选的答案关联对应的题目」。弹窗内显示已有题目列表，用户可选择题号进行关联。底部有「取消」和「关联」按钮。

#### 操作说明

1、用户在左侧资料区框选答案区域后，自动弹出关联弹窗。
2、用户从题目列表中选择要关联的题号。
3、答案框只关联父题，识别后再按子题标记自动拆到子题。
4、点击「关联」后，系统对该答案框进行识别，提取答案和解析内容关联到所选题目。

- 后续流程：关联关系存储在框数据中，识别完成后答案和解析写入对应的题目

- 异常边界：关闭弹窗和点击取消按钮时不保存关联，框也取消框选

#### 验收标准

1. 弹窗正确显示题目列表
2. 选择题号后关联按钮可用
3. 关联后答案正确写入题目
4. 答案只关联父题

### REVIEW_STEP-006 图片模式与编辑模式切换

来源：代码事实

#### 显示说明

右侧面板顶部有「图片模式」和「编辑模式」两个切换按钮。图片模式下每道题显示题目截图；编辑模式下显示AI识别的文字内容，可进行编辑。编辑模式下显示提示标签「请注意甄别AI识别内容」。切换时有加载过渡动画。

#### 操作说明

1、点击「图片模式」切换到图片视图，查看题目原始截图。
2、点击「编辑模式」切换到文字视图，可编辑AI识别的题目内容、答案和解析。
3、切换时显示短暂加载动画。
4、当用户在图片模式下对图片进行了裁剪操作后，切换到编辑模式，再切回图片模式时，显示的是用户裁剪后的图片数据。同理，用户在编辑模式下编辑了文字数据后切换到图片模式，再切回编辑模式时，回显用户之前编辑过的文字数据。清空用户编辑的图片和文字数据的时机是用户重新上传了资料。

#### 验收标准

1. 两种模式可切换
2. 切换有加载动画
3. 编辑模式可编辑内容
4. 图片模式显示截图

### REVIEW_STEP-007 题目图片裁剪功能

来源：代码事实

#### 显示说明

每道题目卡片在编辑模式下都有「使用图片」开关，在图片模式下（无论是全局为图片模式，还是编辑模式下部分题型使用了图片）都有裁剪按钮。开启「使用图片」后显示题目图片，用户可点击裁剪按钮进入裁剪模式。裁剪模式下图片四周显示调整手柄，底部显示「取消」和「确认裁剪」按钮。

#### 操作说明

1、在编辑模式下开启「使用图片」开关后显示题目图片。
2、在图片模式下，点击裁剪按钮进入裁剪模式，拖动四周手柄调整裁剪范围。
3、点击「确认裁剪」保存裁剪结果。
4、点击「取消」退出裁剪模式。
5、裁剪后可点击「还原」恢复到原始图片。

- 后续流程：裁剪后的图片数据保存在题目对象中，加入试卷时传递给下游

#### 验收标准

1. 编辑模式下有使用图片开关
2. 图片模式下有裁剪按钮
3. 确认裁剪保存结果
4. 取消不保存
5. 可还原到原始图片

### REVIEW_STEP-008 加入试卷按钮

来源：代码事实 + 确认决策

#### 显示说明

按钮位于弹窗右上角，文案为「加入试卷」。有题目时绿色高亮，无题目时置灰不可点击。

#### 操作说明

1、点击「加入试卷」后，检查是否有题目未匹配答案解析。
2、如有缺失，弹窗确认：「仍有 X 道题未匹配答案解析，是否继续加入试卷？」（X道题的判断标准：只要该道大题存在答案或解析未匹配，就计1道）。
3、确认后，将题目数据传递给下游试卷编辑页面，完整保留父题、子题、子题答案/解析、填空空位答案、题型、选项数和题目图片。
4、加入试卷前保存当前弹窗状态，以便从试卷编辑页返回时恢复。

- 后续流程：题目数据传递给试卷编辑页面；弹窗状态暂存以供从试卷编辑页返回时恢复

- 异常边界：如果有未匹配的答案解析，弹窗二次确认后才执行加入操作

#### 验收标准

1. 有题目时按钮可用
2. 无题目时按钮置灰
3. 有缺口时弹窗确认
4. 加入后数据完整传递

### REVIEW_STEP-009 识别失败降级处理

来源：代码事实 + 确认决策

#### 显示说明

生产环境下识别失败时，停留在框选识别阶段，不进入核查题目信息页面。显示错误提示「识别失败，请调整框选范围后重试」。

#### 操作说明

1、AI识别失败时，不生成模拟数据，不进入核查页面。
2、用户看到错误提示后，可调整框选范围重新识别。
3、仅识别题目模式下，如果框选区域包含答案/解析，不单独处理，由用户在核查页面通过裁剪功能手动处理。

- 异常边界：识别失败时停留在框选识别阶段，需用户手动重试

#### 验收标准

1. 识别失败停留在框选阶段
2. 显示错误提示
3. 不生成模拟数据
4. 可重试

### REVIEW_STEP-010 答案/解析自动拆分规则

来源：确认决策

#### 显示说明

解答题、完形填空等复合题，答案和解析按子题标记自动拆分到对应子题。无法拆分时保留在父题答案/解析区，并显示「答案解析待人工拆分」的提示。拆分后不保留原始文本，只保留拆分后的子题答案/解析。填空题型按空位自动拆分答案。

#### 操作说明

1、答案/解析只允许按明确子题标记拆分。
2、无法拆分时保留在父题区，标记「答案解析待人工拆分」。
3、拆分后不保留原始未拆分文本。
4、拆分逻辑基于子题数量（≥2个）和子题标记。

- 后续流程：拆分后的答案/解析写入对应子题对象中

- 异常边界：无法拆分时保留在父题区

#### 验收标准

1. 按子题标记正确拆分
2. 无法拆分时显示提示
3. 拆分后不保留原始文本
4. 填空按空位拆分

### REVIEW_STEP-011 全局匹配答案按钮（分步模式）

来源：代码事实

#### 显示说明

识别题目和答案模式下，框选识别步骤中，已有题目且尚未匹配答案时，工具栏下方显示「全局匹配答案」按钮。按钮为靛蓝色文字样式。点击后系统从整页资料中自动定位答案区域并关联到已有题目。

#### 操作说明

1、点击「全局匹配答案」后，系统将整页资料发送给AI，从资料中自动定位答案区域并匹配到已有题目。
2、匹配过程显示进度提示。
3、匹配完成后统计匹配结果。

- 后续流程：将整页图片和已有题目信息发送给AI匹配服务

- 异常边界：匹配过程中按钮不可点击

#### 验收标准

1. 按钮在正确条件下显示
2. 点击后触发匹配
3. 匹配结果正确统计
